使用hexo+github page建立部落格

昨天晚上花了3-4個小時最後終於搞定hexo+github建立部落格
主要是參考這篇文章的教學
部落格的編寫是使用markdown語法，一樣同位博主有簡易的教學

詳細的建立步驟就不贅述了，大致紀錄一下流程還有我操作時遇到的問題和解決方式

順帶一提系統是win10

1.註冊github帳號，設定repository

2.安裝Node.js,git

3.安裝好Node.js後，可以用npm安裝hexo

$ npm install -g hexo-cli
這個步驟可以使用git bash做，也可以用cmd來做(win10中shift+右鍵是開啟powershell)
我其實也不太清楚應該要安裝在哪個路徑比較好，我自己是在存放部落格的資料夾內按右鍵或shift+右鍵，
選擇git bash或powershell下指令來安裝。

4.安裝好hexo就可以開始製作部落格了，一樣在存放資料夾內按右鍵或shift+右鍵來下指令

先初始化部落格
hexo init blog

然後生成
hexo generate
hexo server這個指令是在本地開啟伺服器，在瀏覽器輸入http://localhost:4000/就可以看到部落格

5.上傳部落格到git上

首先打開git bash，要輸入github的使用者名稱和電子信箱
git config --global user.name "使用者名稱"
git config --global user.email "電子信箱"

如果要清除使用者的話，可以輸入
git config --global --unset user.name
git config --global --unset user.email

再來是建立ssh key，輸入
ssh-keygen -t rsa -C "信箱地址"

就可以上傳部落格到git上，過程會叫你登錄github

但是這個方法的缺點是，以我遇到的狀況來說，因為我第一個github帳號被flagged掉，所以即使上傳了部落格仍無法使用github page。
當我換新帳號重新上傳時，git顯示的錯誤訊息仍是我的舊帳號無法上傳內容到新帳號的repository上，雖然我前面已經unset使用者名稱和信箱了。反覆嘗試很多方法，最後我把windows系統還原然後重做一次，仍然無法上傳到新帳號的repository上，我猜可能是git login的問題。

第一篇文章

今天學習如何使用github page + hexo 建立blog，
雖然之前已經找到一篇完整教學，
但由於原本的github帳號被flagged掉，
所以即使成功上傳好blog也無法使用github page。
當我要更換新帳號架設blog時卻發現git只能使用舊帳號來上傳資料…
所有流程全都用新帳號來設定，使用git上傳時卻一直顯示舊帳號沒有權限上傳到新帳號的repo
無奈只好先用VM跑一次流程確認沒有問題，
然後再使用系統還原本系統，
最後再按圖索驥一次，終於成功了!
